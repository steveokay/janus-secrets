# Nocturne N5 — Settings hub + OIDC login button — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Build the `/settings` hub (Instance, OIDC provider, CI federation, Appearance) replacing the current Placeholder, and add an OIDC "Sign in with SSO" button to the login page. All backend endpoints already exist — this phase is frontend-only.

**Architecture:** A `SettingsPage` with a left subnav (URL-driven via `useSearchParams`) and four section components, each a focused file under `web/src/settings/`. **Tech Stack:** React + TS + Vite + Tailwind (Nocturne CSS-var tokens) + TanStack Query + react-router v6 + Vitest/RTL/MSW v2. Kit primitives in `web/src/ui/`.

## Invariants (apply to EVERY task — do not violate)

- **Design tokens only.** No hex, no `dark:` variants, no raw palette classes (`gray-`/`blue-`/`red-`/etc.), no `text-brand-deep`. Compose from Nocturne tokens (`bg-surface-1/2/3`, `text-ink`/`-hi`/`-body`/`-mute`/`-faint`, `border-line`/`-brand-line`/`-danger-line`, `shadow-elev-1`, `shadow-glow`/`-glow-soft`, `bg-row-hover`, `transition-nocturne`, `rounded-card`/`-bar`/`-pill`, `bg-brand-grad`, `text-on-brand`, `bg-brand-soft`, `text-brand-text`, `bg-nav-active`). - **Both themes.** Every screen must render in light + dark; `npm run smoke` checks both.
- **SECURITY — re-verify each task:**
  - The OIDC **`client_secret` is write-only**: the backend never returns it (only `secret_set: boolean`). NEVER prefill it into an input. On edit, an empty secret field means "leave unchanged" — only include `client_secret` in the PUT payload when the user typed a new value. On first configuration (404 state) the secret is required.
  - **Backup** (`GET /v1/sys/backup`) streams an encrypted ndjson dump — it contains only sealed/encrypted material, never plaintext secrets. Still treat it as sensitive: fetch it as a blob, trigger the download, and `URL.revokeObjectURL` immediately. Do not cache it in React Query.
  - **Seal** is destructive (takes the instance offline). Gate it behind a typed-confirm modal (user types `SEAL`).
  - Command palette stays **names-only** (do not add any secret-bearing command). No secret value in logs, toasts, or audit.
- **403/404 handling:** OIDC + federation section content probes its `GET`. `403` → render a "Requires instance admin" hint instead of the content (section stays listed, mirrors ops-console tolerance). `404` (`not_found`) → "not configured yet" empty state with a Configure/Add affordance. Any other error → generic error state.
- **Instance-section caveat (deliberate):** `seal-status` is unauthenticated, so the Instance section is always visible. `POST /v1/sys/seal` (needs `SysSeal`) and `GET /v1/sys/backup` (needs `SysBackup`) have no cheap capability probe, so those two actions are always shown and surface a 403 via `errorMessage`/toast if unauthorized. Document this with a code comment; do not conflate their caps with `OIDCManage`.
- **Data layer:** add endpoints to `web/src/lib/endpoints.ts`; msw mocks in tests MUST mirror the Go wire shapes exactly (field names, status codes, error `code`s).

```ts
import { ApiError } from './api'

// Streams GET /v1/sys/backup as a blob and triggers a download. The dump is
// encrypted/sealed material only — no plaintext secrets — but still sensitive:
// we never cache it and revoke the object URL right after the click.
export async function downloadBackup(): Promise<void> {
  const res = await fetch('/v1/sys/backup', { credentials: 'include' })
  if (!res.ok) {
    const text = await res.text().catch(() => '')
    let code = 'error', message = res.statusText
    try { const e = (text ? JSON.parse(text) : undefined)?.error; if (e) { code = e.code ?? code; message = e.message ?? message } } catch { /* non-JSON */ }
    throw new ApiError(res.status, code, message)
  }
  const blob = await res.blob()
  const cd = res.headers.get('Content-Disposition') ?? ''
  const name = /filename="?([^"]+)"?/.exec(cd)?.[1] ?? 'janus-backup.jsonl'
  const url = URL.createObjectURL(blob)
  try {
    const a = document.createElement('a')
    a.href = url; a.download = name
    document.body.appendChild(a); a.click(); a.remove()
  } finally {
    URL.revokeObjectURL(url)
  }
}
```
