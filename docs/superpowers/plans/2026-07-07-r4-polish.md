# R4 — Screen Polish Pass Implementation Plan

> **For agentic workers:** Small, objective polish slice executed controller-inline (changes are fully specified and low-risk), gated by tests + a final adversarial review. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Close the accumulated dark-redesign polish debt: migrate sub-AA `text-brand-deep` foregrounds to `text-brand-text`, fix the R3 project-board minor items (cycle-safe config tree, env-loading skeleton, page heading), and harden the ⌘K palette's accessibility.

**Architecture:** Token/foreground swaps + targeted component fixes. No new screens, no API changes. Every change must render correctly in BOTH themes.

**Tech Stack:** React 18 + TS + Tailwind (CSS-var tokens) + Radix + Vitest/MSW.

**Scope note:** R4 as spec'd also says "verify each screen (editor, version history, audit, tokens, members, auth/unseal) renders in dark & matches the mockup." Because every component already uses semantic tokens (dark works app-wide), that verification is folded into the final dark-render audit + smoke rather than open-ended restyling. Any genuinely subjective restyle needs surface to the user with concrete artifacts, not guessed.

### Task 1: Dark-AA foreground migration (`text-brand-deep` → `text-brand-text`)

`--brand-text` is `#5546E0` in light (identical to `--brand-deep`, so light is unchanged) and `#A79CFF` in dark (readable, vs `#5546E0` which fails AA on near-black). Pure win in both themes.

**Files:** `web/src/ui/Pill.tsx`, `web/src/ui/EmptyState.tsx`, `web/src/shell/UserMenu.tsx`, `web/src/secrets/SecretEditor.tsx`; new guard test `web/src/test/dark-aa.test.ts`.

- [ ] Replace every foreground `text-brand-deep` and `hover:text-brand-deep` with `text-brand-text` / `hover:text-brand-text` in the four files:
  - `Pill.tsx:11` — `brand: 'bg-brand-soft text-brand-deep'` → `text-brand-text`.
  - `EmptyState.tsx:17` — icon bubble `text-brand-deep` → `text-brand-text`.
  - `UserMenu.tsx` — 4 sites: two `data-[highlighted]:text-brand-deep`, avatar `text-brand-deep`, `ItemIndicator` `text-brand-deep` → all `text-brand-text`.
  - `SecretEditor.tsx` — `overridden: 'bg-brand-soft text-brand-deep'` → `text-brand-text`; two `hover:text-brand-deep` → `hover:text-brand-text`.
- [ ] Guard test `web/src/test/dark-aa.test.ts`: read all `web/src/**/*.tsx` and assert none contain `text-brand-deep` (foreground use is banned; the token still exists for backgrounds if ever needed). Mirror the file-walking style of `web/src/test/no-raw-palette.test.ts`.
- [ ] Run `npx vitest run` (green) + `npm run typecheck`. Commit: `feat(web): migrate brand-deep foregrounds to brand-text for dark AA`.

### Task 3: ⌘K palette accessibility hardening

**Files:** `web/src/palette/CommandPalette.tsx`; test `web/src/palette/CommandPalette.test.tsx` (extend).

- [ ] **Clear query on reopen:** add `useEffect(() => { if (open) setQuery('') }, [open])` so a stale search doesn't persist across opens.
- [ ] **IME guard on Enter:** in `onKeyDown`, ignore Enter while composing — `if (e.key === 'Enter' && !e.nativeEvent.isComposing) { … }` (leave composing keystrokes to the IME).
- [ ] **`aria-activedescendant`:** give each option a stable id `palette-opt-${flatIndex}`; set `aria-activedescendant={filtered.length ? 'palette-opt-' + active : undefined}` on the `role="combobox"` input.
- [ ] **Group semantics:** on each group `<div>`, add `role="group"` + `aria-label={group}`.
- [ ] Test: extend `CommandPalette.test.tsx` — (a) reopening the palette clears a prior query; (b) the active option's id matches the input's `aria-activedescendant`. Run vitest + typecheck. Commit: `fix(web): palette a11y — activedescendant, groups, IME + reopen-clear`.
