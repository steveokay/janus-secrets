# Nocturne N4 — Admin/Ops Surface Restyle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land phase N4 of the Nocturne redo (`ui-redo.md` §4.4, §5): restyle the audit / tokens / members / transit / operations surfaces to the Nocturne token language, add sticky table headers, glow the chain-verified badge, make the operations `last_error` expandable inline, and fold the legacy `timeAgo` formatter onto `relativeTime` (N2 follow-up F4). Presentation-only — no behavior change except the two sanctioned improvements (last-error expand; timeAgo→relativeTime future-date correctness).

**Architecture:** Each surface's legacy alias classes (`text-muted`/`text-faint`/`bg-card`/`shadow-card`/`border-line-soft`/`text-white`) migrate to the N1 ink/surface/on-brand tokens; raw action `<button>`/`<input>` elements swap to the kit `Button`/`Input` primitives where the API fits (the N3 pattern), and four real `<table>`s + the shared `OpsTable` gain a sticky header. One shared code change: delete `web/src/lib/time.ts` (`timeAgo`) and repoint its 3 call sites to `relativeTime`. Recon (main @ `a7b8ff7`) confirmed **no test asserts CSS classes or `timeAgo`'s exact output**, so the suite is a pure behavior gate.

**Tech Stack:** React + TypeScript + Tailwind CSS-var tokens + TanStack Query + Vitest/RTL/MSW v2.

**Sources:** `ui-redo.md` §4.4; spec `docs/superpowers/specs/2026-07-12-nocturne-design.md`; mockup `docs/design/ui-nocturne-mockup.html`. Recon facts (files/lines/aliases) baked into each task below.

## Invariants (every task)

- No hex outside `web/src/theme.css`; no `text-brand-deep`; no `dark:` variants; no palette classes (`gray-`/`blue-`/etc.); token classes only.
- **Security (touch surfaces that render tokens/creds):** `RevealOnce` (token reveal, member initial-password) and `IssuedCredsModal` (dynamic creds) keep their once-only/ephemeral behavior — className edits only, never logic. Masked/list views render names/counts/paths only — never a secret value. The transit Playground's plaintext-free posture (no decrypt/datakey exposure) stays as-is.
- All ~276 tests pass untouched (none assert classes or timeAgo strings — recon-verified). New tests only where this plan adds them (T1).
- Per-task gate: `cd web && npm test -- --run && npm run typecheck`, then commit. Dual-theme smoke on the final sweep (T7).
- **Sticky-header pattern (CORRECTED during T2 review — critical):** the scroll container is `AppLayout`'s `<main className="overflow-y-auto">`. For a `sticky top-0` header to pin, there must be **no `overflow: hidden/auto/scroll` ancestor** between the header and `main`. The audit/tokens/members tables carry `overflow-hidden` (for rounded-card corners) — that DEFEATS sticky, so you MUST **remove `overflow-hidden` from the table card** when adding the sticky header (keep `rounded-card border`; the header's `bg-surface-1` corner bleed at .02 alpha is imperceptible). Header `<tr>`: `sticky top-0 z-10 bg-surface-1`. **Exception — the ops `OpsTable` has `overflow-x-auto` (needed for its `min-w-[720px]` wide table); that horizontal-scroll container conflicts with vertical sticky and can't be cheaply resolved — so DO NOT add sticky there (T6): restyle the header's tokens but leave it non-sticky, noted.** Pre-existing: `SecretTable.tsx` (N3) also has dead sticky classes under an `overflow-x-auto` — out of N4 scope, tracked as a follow-up.
