# Nocturne N3 — Secret Editor Restyle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land phase N3 of the Nocturne redo (`ui-redo.md` §4.3, §5): the flagship secret editor plus its version-history/diff/import surfaces restyled to the Nocturne language — layered table card, per-change row washes + rails, glowing floating save-bar, kit-primitive buttons/inputs, and a new header metadata line — with **zero behavior change**. Two small N2 follow-ups ride along (composed HomePage test; glyph spec carve-out + `--glyph-d` family swap).

**Architecture:** Pure presentation pass over `web/src/secrets/` + `VersionHistory`: legacy alias classes (`text-muted`/`text-faint`) migrate to ink-scale tokens, raw `<button>`/`<input>` elements swap to the N1 kit primitives (`Button`, `Input`, `Textarea`) where the API fits, and three small token additions (row-hover tint, added/removed washes, combined save-bar glow shadow) land in `theme.css`/`tailwind.config.js` so no component ever carries a raw color. The metadata line renders from data the editor already fetches (`listVersions` + masked rows). Recon (2026-07-13) confirmed **no editor test asserts classes**, so the suite is a pure behavior gate.

**Tech Stack:** React + TypeScript + Tailwind CSS-var tokens + TanStack Query + Vitest/RTL/MSW v2.

**Sources:** `ui-redo.md` §4.3 (treatment), `docs/superpowers/specs/2026-07-12-nocturne-design.md` (tokens), `docs/design/ui-nocturne-mockup.html` §editor (visual reference). Recon facts below are from main @ `ba94ae1`.
